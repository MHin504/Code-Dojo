Coding Dojo 24.05.2021

Kata Gilded Rose

Aufgabe
- 2 Teams bilden â 3 Personen
- Neue Funktionalität integrieren und sicher stellen, dass alte Funktionalität erhalten bleibt.

Ziele:

- 3 Minuten Zeitlimit um einen Test zu schreiben (ohne Ein- und Auschecken)
- Testabdeckung herstellen
- Refaktoriseren
- Neue Funtionalität einbauen

Vorgehen:
- Prop für Items Field einführen, damit wir das Ergebnis der updateQuality Methode zu prüfen
- Tests schreiben, welche die Beschreibung der Aufgabe prüfen (möglicherweiße passt dieser nicht zum Code)
- Wenn Tests rot werden => Annahme anpassen
- Anschließend Test an die Wahrheit anpassen

Was wird wie refaktorisiert:
- Erst Tests schreiben wie oben beschrieben
- Code in 2 Klassen auslagern
	- Item
	- InventorySystem (beinhaltet Methode UpdateQuality)
	- InventorySystem aus Main aufrufen
- R# Refaktorisierungen durchführen
- Renamings
- MagicNumbers auflösen => Konstanten einführen
- DRY anwenden
- Versuchen UpdateQuality in kleinere private Methoden aufzusplitten
